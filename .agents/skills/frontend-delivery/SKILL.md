# Frontend Delivery

Use this skill for general frontend work when no more specific project skill applies.

## What to do

1. Inspect the existing UI structure, state flow, styles, and shared components first.
2. Preserve the project’s visual language unless the user asks for a redesign.
3. Prefer clear layout structure, responsive behavior, and explicit states.
4. Keep data flow and side effects easy to trace.
5. Verify the changed UI on the most relevant desktop or mobile path.

## Guardrails

- Do not introduce generic UI churn.
- Do not replace established patterns without a reason.
- Keep visual changes intentional and consistent.
