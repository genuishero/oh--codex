# Ship Changes

Use this skill when the task is action-oriented.

## What to do

1. Inspect before editing.
2. Choose the smallest correct change.
3. Match existing patterns before introducing anything new.
4. Verify with the most relevant diagnostics, tests, or builds.
5. Report what changed and what evidence supports completion.

## Guardrails

- Do not stop at analysis when the request clearly asks for changes.
- Do not skip verification.
- Keep scope tight.
