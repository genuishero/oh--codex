# Plan First

Use this skill when the user wants thinking before changes.

## What to do

1. Clarify the real goal.
2. Break the work into the smallest useful phases.
3. Call out tradeoffs, risks, and assumptions.
4. Keep the plan executable rather than abstract.
5. Do not start editing unless the user clearly switches to implementation.

## Output shape

- Goal
- Recommended approach
- Step breakdown
- Risks or decisions
